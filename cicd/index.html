项目概述 生产级 · 已落地

基于 Jenkins Pipeline 构建的端到端 CI/CD 自动化部署流水线， 支持 多分支、多环境、 多项目 的自动化编译、测试、SonarQube 分析、Docker 镜像构建与推送、 Kubernetes 滚动部署，并集成 企业微信通知 与 覆盖率报告。

多分支支持 SonarQube 质量门禁 Docker + K8s 部署 企业微信通知 覆盖率报告归档
核心能力 6 大特性
📦 多项目构建 支持后端（Maven）与前端（Vue/NPM）混合构建
🔀 多分支策略 dev/test/prod 分支独立部署，环境隔离
📊 SonarQube 分析 代码质量门禁，Bug/漏洞/异味实时检测
🐳 Docker 镜像 标准化镜像构建，推送到私有仓库
☸️ Kubernetes 部署 kubectl 滚动更新，零停机发布
📱 企业微信通知 构建状态实时推送，覆盖率/质量报告
部署架构图 CI/CD 端到端自动化部署架构
Git → Webhook → Jenkins → Build → Test → Sonar → Docker → Push → K8s → Notify
构建参数 6 个参数
参数名 类型 选项/默认值 描述
CUSTOM_BRANCH Choice main / feature/* / release/* 选择分支
CUSTOM_PROJECT String ''（必填） 项目名称
CUSTOM_ENVIRONMENT Choice dev / test / prod 选择部署环境
CUSTOM_VERSION String ''（留空用分支名） 自定义版本号
SKIP_TESTS Boolean false（默认跳过） 是否跳过测试
SKIP_DEPLOY Boolean false（默认部署） 是否跳过部署
流水线步骤 8 个阶段
① 环境配置
1 参数解析与校验（分支/项目/环境/版本） 初始化
2 环境变量注入（JDK/Maven/镜像仓库/Sonar） 配置
② 拉取代码
3 Git 拉取指定分支代码（含凭据认证） Git
③ 编译构建
4 后端：Maven clean install（可跳过测试） Maven
5 前端：npm install + build（环境差异化构建） NPM
④ 单元测试与覆盖率
6 后端：mvn test + JaCoCo 覆盖率报告 JaCoCo
7 前端：npm run test:unit --coverage Jest
⑤ SonarQube 分析
8 SonarScanner 代码质量扫描（Bug/漏洞/异味/覆盖率） SonarQube
⑥ 镜像打包与部署
9 Docker 镜像构建（Dockerfile） Build
10 镜像推送到私有仓库（docker login + push） Push
11 Kubernetes 滚动部署（kubectl set image + rollout） Deploy
⑦ 覆盖率报告归档
12 归档 JaCoCo / Jest 覆盖率报告（HTML + lcov.info） Archive
⑧ 构建后通知
13 企业微信通知（成功/失败/不稳定/取消） Notify
Pipeline 核心代码 Declarative Pipeline 
pipeline {
    agent any

    // ===== 构建参数 =====
    parameters {
        choice(name: 'CUSTOM_BRANCH', choices: ['dev', 'test', 'mes_1'], description: '选择分支')
        string(name: 'CUSTOM_PROJECT', defaultValue: '', description: '项目名称')
        choice(name: 'CUSTOM_ENVIRONMENT', choices: ['dev', 'test'], description: '部署环境')
        string(name: 'CUSTOM_VERSION', defaultValue: '', description: '版本号（留空用分支名）')
        booleanParam(name: 'SKIP_TESTS', defaultValue: true, description: '跳过测试')
        booleanParam(name: 'SKIP_DEPLOY', defaultValue: false, description: '跳过部署')
    }

    // ===== 环境变量 =====
    environment {
        CHAT_WEBHOOK_KEY = credentials('wechat-webhook-key')
        GIT_CRED_ID       = 'fa2845e8-57d8-461b-9fb4-63cb5a285752'
        DOCKER_CRED_ID    = 'docker-registry-cred'
        SONAR_TOKEN       = credentials('sonar-qube-token')
        IMAGE_REGISTRY    = 'dev.klhbs.com:8433'
        SONAR_URL         = 'http://dev.klhbs.com:9000'
        SONAR_SCANNER_HOME = tool(name: 'Sonar-Scanner-4.x')
    }

    // ===== 流水线阶段 =====
    stages {
        stage('环境配置') { steps { script { ... } } }
        stage('拉取代码')   { steps { checkout(...) } }
        stage('编译构建')   { steps { sh 'mvn clean install' } }
        stage('单元测试与覆盖率') { when { !params.SKIP_TESTS } steps { ... } }
        stage('SonarQube 分析') { when { !params.SKIP_TESTS } steps { ... } }
        stage('镜像打包与部署') {
            when { !params.SKIP_DEPLOY }
            steps {
                sh 'docker build -t ${PROJECT}:${VERSION} .'
                sh 'docker push ${IMAGE_REGISTRY}/library/${PROJECT}:${VERSION}'
                sshPublisher(...)  // kubectl set image + rollout
            }
        }
        stage('覆盖率报告归档') { when { !params.SKIP_TESTS } steps { ... } }
    }

    // ===== 构建后操作 =====
    post {
        always    { cleanWs() }
        success   { sendNotification("成功", "info") }
        failure   { sendNotification("失败", "danger") }
        unstable { sendNotification("不稳定", "warning") }
    }
}
